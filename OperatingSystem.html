    • What is Operating System ?

    What is O.S.? :- An O.S. is a program that manages application programs and acts as an intermediary b/w the computer user and the computer hardware.The purpose of operating system is to provide an environment in which a user can execute program in a convenient and efficient manner.
    Operating system is a system software with provide an environment to help the user to execute the programs. The operating system is a resource manager which allocates and manages various resourses like processes, main memory, input/output device and information on secondary storage devices.

    Functions of Operating system :-

    Operating system perform the following system.

    • Process Managment :- It means assining processor to different task which has to be performed by the computer system.
    • Memory Managment :- It means a location of main memory and secondary storage area to the system programs as well as user programs and data.
    • Input and Output Managment :- It means coordination assignment of different output and input devices while one or more programs are executed.
    • File system Managment :- Operating system also responsible for maintanance of a file system, in which the users are allowed to created, delete and move files.
    • Establishment and Enforcement of a priority system:- It means the operating system determines and maintains the order in which jobs are to be executed in the computer system.
    • Assignment of system resourses. Both software and hardware to the various user of the system.

    Evolution of operating system (Types of Operating System) :-

    A brief description of the Evolution of operating system has been described below.

    1. Simple batch operating system :- In the earliest bay digit computer usually runs from a cansole. Input/Output devices cansisteb of card reader, tape drives and lind printers. Direct user intervention with the system did not exist. User made a job wiht cansisting of programs. Data and control information. The job was submitted to an operator who would execute the job on the computer system. The output appeared after minutes hours or sometimes days. The user collected the output from the operator. Which also include a memory dump. The operating system was very simple and its major task was to be transfer control from one job to another. The operating system was resident in memory. To speed up processing, jobs with the same needs were bached together are executed as a group. For example, all FORTRAN as a group. Together for execution, thus batch operating system come into existence. Even though processing speed increased to large extent becouse of batch processing. The CPU was often idle. This is becouse of the disparity between operating speed of electronic device like the CPU and the machanical input output devices. CPU operates in the micro-second or nano-second range where as input output devices work in the second or minutes range. To overcome this problem the concept of spooling came into picture. Instead of reading from slow input devices like card reader into the computer memory and then processing the job, the input is first read into the disk. When the job is processed or executed. The input is read directly from the disk. Similarly when a job is executed for printing, the output is written into a buffer on the disk and actually printed later. This form of prossing is known as spooling. Spooling uses the disc as a large buffer to read ahead as possible on input devices and for storing output untill output devices are available to accepts them. Spooling increases the performance of the system by alloeing both a faster CPU and slower input output devices to work at higher operating rate.
    2. Multi-Programmed batch operating system :- Job scheduling bring in the ability of multiprogramming. A single user cannot keep both CPU and input device busy. Multiprogramming increases CPU untilisation by organising jobs. Choosing one out of several ready jobs in memory for execution by the CPU is called the CPU scheduling.
    3. Time sharing operating system :- CPU scheduling and multiprogramming provide each user one time slice in a time shared system. A program in execution is reterred to as a process. A process execute for one time slice at a time. A process may need more than one time slice to complete. During a time slice a process may finish execution or go for an input output. The CPU switches between processes at the end of a time slice. The switching between processes is so fast that the user gets the illusion that the CPU is rxecuting only one user's process.
      Time sharing operating system are more complex than multiprogrammed operating system becouse in multiprogramming several jobs are must be kept simultaneously in memory. Which requires some form of memory managment and protection. Jobs we have to be swapped in and out of main memory to the secondary memory. To achive this goal a techanique called virtural memory is used.
      Multiprogramming and time sharing are the main concepts is all modern operating system.
    4. Personal computer operating system :- The main mode of a user PC is by a single user. The operating system for PCs were designed as a single task operating system.
      The operating system of PCs consists of two parts. One part is called the basic BIOS which is stored in ROM. The other part is stored in floppy disk or hard disk. When power is turned on BIOS takes control. It does what is known as POST. The test sees whether memory is ok and all other relevant units function. Having done this, it reads from the disk a small portion of O.S known as the boot and load it into the memory. This boot program then "pulls" the rest of the operating system from the disk and store it in the main memory. This is known as "booting the system".
    5. Multiprocessor operating system (Parallel system) :- Multiprocessing system will be having more than one processor and all these processor will share the computer bus, the clock and sometimes memory and peripheral devices. These system becouse the processes share the memory or a clock.
      There are several resons for building such systems. One advantage is increased throughput. By increasing the number of processors we can get more work done in a shorter time. Another advantages is, it can save money compared to multiple single systems becouse the processes can share peripherals, cabinates and power supplies. Another advantage is that they increase reliability. If functions can be distributed properly among the several processor than the failure of one processor will not halt the system but rather will only slow it down.
      6. There are two most common type of Multiprocessor system :-
      1. Symmetric multiprocessor model, in which each processor runs and identical copy of the operating system and these copies communicates with one another as needed.
      2. Asymmetric multiprocessor model, in which is processor is assigned a specific task. Here one processor serves as master which control the entire system and all the other processes serve as slaves. This model represent a master-slave relationship.
    6. Distributed system :- In this type of system all the computations are distributed among several processes. distributed system also referred as loosely coupled systems becouse the processor do not share the memory or a clock. Each processor has its own local memory. The processor communicate with one another through various communication lines, such as high speed buses or telephone lines. The processors in distributed system vary in size, function and referred as sites, nodes, computers and so on, depending on the context in which they are mentioned.
      7. Reasons for building distributed system :-
      • Resource sharing :- If different user are connected to one another with different resources then the user at one site may be able to use the resources available at another.
      • Computation speedup :- In distributed system a particular Computation will be partitioned into number of sub computations that can run concrrently. in addition to this, distributed system may allow us to distributed the computations among various sites. it is called load sharing.
      • Reliability :- it means, if one system fails, the remaining site will share the work of failed site.
      • Communication :- When many side are connected to one another by communication network. The process at different sites have the opportunity to exchange information. A user may initiate file transfer ar communicate with one another via email.
    7. Real time system :- Real time operating system are specialy disigned to respond to event that happens in real time. it has well defined, fixed time constants. Processing must be done within the define constants or else the system will fail. This feature is very useful in implementing systems such as airlines reservation system. There are two types of real time system. A hard time system guaranteen that critial task complete at a specified time. A less restrictive type of real time system, where is critial real time task gets priority over other task and remains that priority until. It completes. The several areas in which this type is useful are multimedia, advanced scientific project. Becouse of the expanded used for soft real time functionality. It is finding its way into most current operating systems including major version of unix and windows NT operating system.
